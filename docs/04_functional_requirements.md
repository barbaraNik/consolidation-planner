### Функциональные требования
| ID    | Требование | Приоритет |
|-------|-------------|------------|
| FR-01 | Система должна отображать список заказов, доступных для консолидации (не входящих в активные консолидации), с возможностью фильтрации по порту. | Must |
| FR-02 | Система должна предоставлять пользователю возможность выбора типа контейнера из справочника container_types. | Must |
| FR-03 | Система должна предоставлять возможность ручного выбора заказов для консолидации. | Must |
| FR-04 | Система должна предоставлять возможность автоматического подбора заказов для консолидации: алгоритм формирует оптимальный набор, максимизируя F-score в пределах ограничений. | Must |
| FR-05 | Система должна выполнять предварительный расчёт загрузки контейнера (PV, PM, F-score) без сохранения в БД. | Must |
| FR-06 | Система должна сохранять консолидацию в БД с фиксацией всех метрик и связанных заказов. | Must |
| FR-07 | Система должна предоставить просмотр отчёта по консолидации с детализацией по заказам. | Must |
| FR-08 | Система должна отображать список сохранённых консолидаций (Истории расчётов). | Should |
| FR-09 | Система должна предоставлять экспорт отчёта в формате PDF или Excel. | Could |
