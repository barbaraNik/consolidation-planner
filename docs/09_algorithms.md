# 09. Алгоритмы автоподбора и расчёта F-score

## 1. Общая логика работы алгоритма

Алгоритм получает на вход набор грузовых мест (коробов) с известными габаритами, весом и допустимыми ориентациями, а также параметры выбранного контейнера. Задача — разместить все грузы внутри контейнера так, чтобы не было пересечений, соблюдалась устойчивость и не превышался лимит по весу. На выходе формируется конкретная схема укладки с координатами каждого короба и итоговые метрики загрузки.

Программа может работать в двух режимах:

- **Автоподбор** — система сама формирует пул заказов-кандидатов и подбирает оптимальную комбинацию грузов для загрузки.
- **Ручная выборка** — пользователь указывает конкретные заказы, система рассчитывает возможную укладку и метрики.

---

## 2. Входные данные

Для каждого грузового места i задаются:

- Базовые размеры: l (длина), w (ширина), h (высота), в метрах
- Вес: m, в килограммах
- Допустимое множество ориентаций R — набор разрешённых поворотов короба

После выбора конкретной ориентации r из множества R получаем ориентированные размеры короба:

    a, b, c  ∈  R_i

где a, b, c — длина, ширина и высота короба после поворота.

Размещение короба задаётся координатами его левого нижнего ближнего угла:

    x_i, y_i, z_i

Тогда короб занимает параллелепипед:

    [x_i, x_i + a_i] × [y_i, y_i + b_i] × [z_i, z_i + c_i]

Для контейнера задаются внутренние размеры L, W, H и допустимая грузоподъёмность M.

---

## 3. Ограничения размещения

### 3.1. Границы контейнера

Каждый короб должен полностью находиться внутри контейнера:

    0 ≤ x_i,    x_i + a_i ≤ L
    0 ≤ y_i,    y_i + b_i ≤ W
    0 ≤ z_i,    z_i + c_i ≤ H

Для контейнера 40DC конкретные значения (в мм):

    x_i + a_i ≤ 12032
    y_i + b_i ≤ 2352
    z_i + c_i ≤ 2395

### 3.2. Непересечение коробов

Любые два короба i и j не должны пересекаться. Условие выполняется, если хотя бы одно из следующих неравенств истинно:

    x_i + a_i ≤ x_j   или   x_j + a_j ≤ x_i
    y_i + b_i ≤ y_j   или   y_j + b_j ≤ y_i
    z_i + c_i ≤ z_j   или   z_j + c_j ≤ z_i

### 3.3. Ортогональность

Коробы располагаются только параллельно стенкам контейнера. Допустимые ориентации определяются атрибутами короба:

- Если can_tilt = FALSE, короб размещается только в исходной ориентации l × w × h.
- Если can_stack = FALSE, никакой другой короб не может быть размещён поверх данного.

### 3.4. Ограничение по весу

Суммарный вес всех размещённых коробов не должен превышать грузоподъёмность контейнера:

    Σ m_i ≤ M

Для контейнера 40DC: M = 26 480 кг.

### 3.5. Устойчивость

Каждый короб либо стоит на полу контейнера (z_i = 0), либо все четыре угла его нижнего основания опираются на верхние грани нижележащих коробов.

Множество коробов-опор для короба i:

    B_i = { j : z_j + c_j = z_i }

То есть это коробы, чья верхняя грань находится ровно на уровне дна короба i.

Нижние углы основания короба i:

    (x_i, y_i),  (x_i + a_i, y_i),  (x_i, y_i + b_i),  (x_i + a_i, y_i + b_i)

Для каждого угла (u, v) должен существовать короб-опора j из множества B_i такой, что:

    x_j ≤ u ≤ x_j + a_j   и   y_j ≤ v ≤ y_j + b_j   и   z_j + c_j = z_i

## 4. Метрики загрузки контейнера

### 4.1. Коэффициент загрузки по объёму (PV)

Отношение суммарного объёма размещённых коробов к внутреннему объёму контейнера:

    PV = Σ(a_i × b_i × c_i) / (L × W × H)


### 4.2. Коэффициент загрузки по весу (PM)

Отношение суммарного веса размещённых коробов к допустимой грузоподъёмности:

    PM = Σ m_i / M


### 4.3. Доля размещённых грузовых мест (PN)

Отношение количества фактически размещённых коробов к общему числу коробов в консолидации:

    PN = N_placed / N_total

### 4.4. Итоговый коэффициент загрузки F-score

Взвешенная сумма трёх метрик:

    F = α × PV + β × PM + γ × PN

Веса коэффициентов:

    α = 0.6  — приоритет объёмной загрузки (так как основная цель - это заполнение контейнера по объему, потому что основной задачей системы является максимально эффективное использование его внутреннего объёма)
    β = 0.3  — учёт веса (чтобы программа не выбирала только легкие, но объемные коробки, потому что необходимо обеспечивать более равномерное распределение массы и эффективное использование допустимой грузоподъёмности контейнера)
    γ = 0.1  — бонус за размещение большего числа грузовых мест

Итоговая формула:

    F = 0.6 × PV + 0.3 × PM + 0.1 × PN

---

## 5. Дополнительные метрики остатка

По объёму:

    V_rest = L × W × H  −  Σ(a_i × b_i × c_i)

По весу:

    M_rest = M  −  Σ m_i

Процент объёмной загрузки:

    LoadV = 100 × PV

Процент весовой загрузки:

    LoadM = 100 × PM
